# 时光列车：驶向强国梦

> 一个四年级小朋友和他的姑爷一起创作的红色爱国主题编程作品

---

## 🎯 项目简介

这是罗田县实验小学四年级（13）班涂明洋小朋友和他的姑爷一起完成的编程作品。学校布置了一份作业，要求我们围绕"红色爱国"主题做一个编程作品。我和姑爷一起，在AI的帮助下，设计并开发了这款互动历史学习游戏。

---

## 💡 我是怎么想到做这个项目的

在课本里，我读到了很多关于中国从积贫积弱走向繁荣富强的故事，比如1921年在南湖红船上中国共产党成立、1949年天安门开国大典、改革开放让祖国腾飞、还有神舟飞船飞向太空...

这些历史年份和事件，老师让我们背下来，可是我觉得光背数字有点枯燥。我想：如果能让同学们像坐上一辆时光列车一样，穿越到那些历史现场，亲眼看到、亲耳听到当时发生的事情，那学习历史会不会更有趣呢？

于是我就想到了做一个"时光列车"的游戏——点击不同的按钮，列车就会开到不同的时代，有画面、有声音、有讲解，就像真的在穿越时空一样！

---

## 🤝 我是怎么向姑爷求助的

我的姑爷是一名专业的程序员，平时工作很忙。有一天放学回家，我跟姑爷说："姑爷，学校要我们做一个编程作业，我想做一个时光列车的历史游戏，可是我只会一点点简单的编程，你能帮我吗？"

姑爷听完我的想法后，眼睛都亮了！他说："这个想法很棒！正好让我来教你什么是真正的编程思维。"

姑爷告诉我，编程不只是写代码，而是要先**构思**——想清楚要做什么；然后**设计**——画出界面、规划功能；接着**规划**——把大任务拆成小任务；再**实施**——一步步写代码；最后**测试**——看看有没有bug，还要**复盘**——总结学到了什么。

---

## 👨‍👦 我们是怎么一起协作的

### 第一步：构思阶段

我和姑爷坐在电脑前，我把想法一股脑儿说出来：
- 要有四个历史场景：红船、开国大典、改革开放、航天
- 要有一个小向导给同学们讲解
- 要有勋章收集，看完一个时代就得一个勋章
- 集齐四个勋章要有一个彩蛋惊喜

姑爷把这些想法记在一个文档里，然后问我："你希望这个游戏看起来是什么风格？"

我说："要可爱一点，像绘本一样的风格，我们小朋友喜欢看！"

### 第二步：设计阶段

姑爷打开一个绘图软件，让我看着屏幕，我们一起讨论：
- 主界面是什么样子？——我想要一辆时光列车，四个按钮像车站一样排列在下面
- 每个场景用什么颜色？——红船用红色、开国大典用金色、改革用蓝色、航天用星空
- 向导长什么样？——我画了一个戴红领巾的小学生，姑爷说很好！

### 第三步：AI生成素材

这一步超级神奇！姑爷说现在的AI可以帮我们生成图片、音乐、甚至说话的声音！

- **图片生成**：我们用谷歌和豆包的AI，输入我的想法描述，AI就画出了漂亮的场景图片——南湖红船、天安门、东方明珠塔、神舟飞船...每一张都像绘本插画一样可爱！

- **音乐生成**：姑爷用MiniMax的AI音乐工具，输入"革命时期庄严的音乐"、"庆祝胜利的音乐"、"太空探索的音乐"，AI就自动作曲给我们！

- **语音合成**：我们把讲解的文字给AI，AI就用一个小朋友的声音把解说词读出来，就像真的有个人在讲故事！

- **代码编写**：姑爷让我在旁边看着，他用GLM这个AI助手帮忙写代码。AI写完一段，姑爷就检查一遍，告诉我哪里好、哪里要改。

### 第四步：编码阶段

姑爷用VS Code这个软件写代码。他说代码就像给电脑写指令，告诉电脑什么时候该干什么。

我坐在姑爷旁边，看着他一行一行地写：
- `main.py` 是游戏的入口，像大门一样
- `scenes/` 里面放着四个历史场景的代码
- `ui/` 里面放着按钮、徽章、对话框这些界面组件
- `audio_manager.py` 负责播放音乐和语音

姑爷一边写，一边给我解释这是什么意思。虽然有些代码我还看不太懂，但我知道了原来一个游戏是这么组装起来的！

### 第五步：测试阶段

代码写完了，姑爷说："现在要测试了，看看有没有bug。"Bug就是程序里的错误，会让游戏出问题。

姑爷运行游戏，让我来玩一遍。我点击红船按钮，画面切换了，音乐播放了，向导开始讲解...一切都很顺利！

但是我发现一个小问题：有时候点击太快，声音会卡住。姑爷说这是"并发问题"，然后就改了几行代码，问题就解决了！

### 第六步：复盘阶段

最后，姑爷让我总结这次学到了什么。我说：
- 学会了编程不只是写代码，要先想清楚要做什么
- 学会了怎么把大任务拆成小任务
- 学会了AI可以帮我们生成图片、音乐、声音
- 学会了测试很重要，要找出bug

姑爷笑着说："很好！你已经有了编程思维的雏形了。"

## 🛠️ 我们用到的AI技术

姑爷告诉我，这次项目我们用了好多AI工具：

| AI工具 | 用途 |
|-------|------|
| MiniMax | 语音合成（把文字变成说话的声音）、音乐生成（自动作曲） |
| 谷歌/豆包 | 图片生成（画出场景、向导、徽章） |
| GLM | 文本生成（帮忙写解说词）、代码编写（辅助编程） |
| VS Code | 编码平台（写代码的软件） |

姑爷特别告诉我一个很重要的道理：**"未来AI会代替人类完成很多具体的工作，但人类始终要将AI作为增强自己的工具。相比技术，更重要的是从小锻炼自己的架构能力、逻辑思维、商业敏感度。"**

这句话我一开始不太懂，姑爷解释说：AI就像一个很厉害的助手，可以帮我们画画、写代码、作曲，但是**要让它画什么、写什么、作什么曲，这些决定还是要靠我们人类**。所以我们要学会：
- **架构能力**：把整个项目想清楚，像盖房子一样先设计好框架
- **逻辑思维**：一步步思考问题，把复杂的事情拆简单
- **商业敏感度**：知道别人喜欢什么、需要什么

姑爷说，这些能力比单纯的技术更重要，以后长大了不管做什么工作都用得上！

## 🚧 我们遇到的难点

做这个项目的时候，我们遇到了好多困难：

### 1️⃣ 大小限制的挑战

学校要求我们把作品上传到"小鹿APP"，但是这个APP有一个很严格的限制：**作品总大小不能超过20MB**！

我们一开始生成的素材加起来有60MB！图片太大、音乐太长、语音文件太多了...

姑爷想了好多办法：
- 把图片分辨率降低，但保持画面清晰
- 把音频文件压缩，用更小的格式
- 把不必要的素材删掉，只保留最重要的

我们试了好多次，最终把60MB压缩到了不到20MB，姑爷说这是"极限压缩"！

### 2️⃣ 丝滑的动画效果

为了让游戏更好玩，姑爷加了好多动画效果：
- 点击按钮时有缩放动画
- 场景切换时有淡入淡出的过渡
- 对话框弹出时有缩放效果
- 徽章点亮时有闪亮特效

这些动画很细微，可能小朋友们玩的时候不会特意注意到，但它们让整个游戏的体验**特别丝滑**，不会觉得卡顿或者突兀。

姑爷说："好的用户体验就是让用户感觉不到技术的存在。"

### 3️⃣ 勋章和彩蛋系统

为了让小朋友们更愿意学完所有的历史知识，我们设计了：
- **勋章系统**：看完一个历史时代，右上角就点亮一枚勋章（觉醒、建国、腾飞、航天）
- **彩蛋场景**：集齐四枚勋章，就会触发隐藏的彩蛋——烟花绽放、《我和我的祖国》音乐响起！

姑爷说这叫"激励机制"，让小朋友们有成就感，愿意把所有内容都看完。

## 🎮 项目功能介绍

这个游戏有：

- **1个主界面**：时光列车出发站，四个时代按钮排列在下方
- **4个历史场景**：每个场景有4个章节的知识讲解
- **1个隐藏彩蛋**：集齐勋章后解锁的庆祝场景

### 四个历史时代

| 场景 | 历史时期 | 核心内容 | 勋章 |
|-----|---------|---------|-----|
| 红船启航 | 1921年 | 中国共产党在南湖红船上成立 | 觉醒徽章 |
| 开国大典 | 1949年 | 中华人民共和国成立 | 建国徽章 |
| 改革春风 | 1978年+ | 经济改革，祖国腾飞 | 腾飞徽章 |
| 航天强国 | 2003年+ | 神舟飞船，太空探索 | 航天徽章 |

### 怎么玩？

1. 打开游戏，看到主界面
2. 点击下方的时代按钮，进入历史场景
3. 看红领巾小向导讲解历史知识（有语音和文字）
4. 点击"下一个"看完所有章节
5. 看完一个时代，获得一枚勋章
6. 集齐四枚勋章，解锁彩蛋惊喜！

│   ├── narrative_bubble.py # 对话框
